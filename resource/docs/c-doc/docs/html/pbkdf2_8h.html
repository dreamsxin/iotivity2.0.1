#define PBKDF_ITERATIONS   1000

The number of iterations desired to derived key.

As specified in the OCF Security Specification. This choice is required for interoperability in the Random PIN OTM.

Function to derive cryptographic key from the password.

(RFC 2898) In this implementation, HMAC with SHA2 is considered as a pseudorandom function

Parameters
passwdis the master password from which a derived key is generated.
pLenis the byte size of the passwd.
saltis a cryptographic salt.
saltLenis the byte size of the salt.
iterationsis the number of iterations desired.
keyLenis the desired byte size of the derived key. (should be the same as derivedKey size)
derivedKeyis the generated derived key
Returns
0 on success
