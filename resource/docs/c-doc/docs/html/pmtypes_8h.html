typedef void(* OCGetCSRResultCB) (void *ctx, size_t nOfRes, OCPMGetCsrResult_t *arr, bool hasError)

Callback function definition of CSR retrieve API.

Parameters
[out]ctx- If user set a context, it will be returned here.
[out]nOfRes- total number of results
[out]arr- Array of OCPMGetCsrResult_t, containing one entry for each target device. If an entry's res member is OC_STACK_OK, then csr and csrLen are valid; otherwise they should not be used. This memory is only valid while the callback is executing; callers must make copies if the data needs to be kept longer.
[out]hasError- If all calls succeded, this will be false. One or more errors, and this will be true. Examine the elements of arr to discover which failed.

Callback function definition of roles retrieve API.

Parameters
[out]ctx- If user set a context, it will be returned here.
[out]nOfRes- total number of results
[out]arr- Array of OCPMGetRolesResult_t, containing one entry for each target device. If an entry's res member is OC_STACK_OK, then chains and chainsLength are valid; otherwise they should not be used. This memory is only valid while the callback is executing; callers must make copies if the data needs to be kept longer.
[out]hasError- If all calls succeeded, this will be false. One or more errors, and this will be true. Examine the elements of arr to discover which failed.

typedef void(* OCProvisionResultCB) (void *ctx, size_t nOfRes, OCProvisionResult_t *arr, bool hasError)

Callback function definition of provisioning API.

Parameters
[in]ctx- If user set his/her context, it will be returned here.
[in]nOfRes- total number of results, it depends on using which provisioning API.
[in]arr- Array of OCPMResult_t, each OCPMResult_t contains result for target Device.
[in]hasError- If there is no error, it's returned with 'false' but if there is a single or more error is/are occured during operation, it will be 'true'.
